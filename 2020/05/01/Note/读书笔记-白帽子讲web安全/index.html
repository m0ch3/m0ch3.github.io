浏览器安全

同源策略

浏览器同源策略，限制了来自不同源的document或者脚本，对当前document读取或者设置某些属性。

对于js，影响同源的因素包括host，子域名，端口，协议，即“同host，同子域名，同端口，同协议”才被认为是同源。

注意：页面加载的js来源域不重要，重要的是加载js页面所在的页面加载域，例如：
a.com通过<script src=http://b.com/b.js></script>加载了b.com上面的b.js，但是b.js是运行在a.com中的，因此对于a.com，b.js的源（Origin）是a.com。类似带有src属性的可远程加载资源的标签，都可以跨域加载，不受同源策略限制，但是浏览器限制了js的权限，使其不能读写返回的内容。

对于XMLHttpRequest来说，它可以访问来自同源对象的内容，但是受到同源策略限制，不能跨域访问资源，在ajax开发中应当注意。

除此还有一些插件会遵从各自的同源策略，如Flash，Java Applet，Silverlight，Google Gears都有自己的控制策略。Flash的控制策略通过目标网站根目录下面的crossdomain.xml来判断是否允许跨域访问。
